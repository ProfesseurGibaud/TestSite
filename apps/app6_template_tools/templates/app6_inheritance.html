Template inheritance

/!\ Look at the source first /!\

Inheritance is a form of template reuse. But it is NOT an include.

In this exemple, the view renders THIS (app6_inheritance.html) template.

But this template inherits from another template named app6_base.html.

It works this way :

  1. This templates (app6_inheritance.html) says which template it inherits from using the extends tag.
  2. This templates define named blocks of content using the block tag.
  3. When the view renders the template, it wraps this (app6_inheritance.html) template in the parent (app6_base.html), filling the parent blocks with the child blocks.

Schematically, it works this way:

Child (app6_inheritance.html) :

         _____
        |_____|

         _____
        |     |
        |     |
        |_____|

when this is rendered, it is wrapped IN the parent (app6_base.html) :

    ................
    .     _____    .
    .    |_____|   .
    .              .
    .     _____    .
    .    |     |   .
    .    |     |   .
    .    |_____|   .
    .              .
    ................

It works backward, compared to an include. In a include, you would render the parent, and insert the child.

You don't need to fill all the blocks of the parent (which can have default values), and you can inherit from and a child, making complex inheritance possible.
